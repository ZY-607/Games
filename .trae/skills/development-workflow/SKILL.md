# 开发工作流程管理

此技能确保在开发过程中始终维护README文档和版本号，方便追踪项目历史和变更。

## 核心原则

### 自动化文档管理
- 每次需求确认后，自动更新README
- 每次代码修改后，自动更新版本号和README
- 版本号显示在工具名称旁边或右下角，不显眼但可见

### 版本号管理
- 版本号格式：v1.0.0 (主版本.次版本.修订号)
- 主版本：重大功能变更或不兼容的修改
- 次版本：新功能添加，向后兼容
- 修订号：bug修复或小的改进

## 使用场景

### 场景1：需求确认后
当用户确认所有需求后：
1. 创建或更新README.md
2. 记录当前版本号（初始为v1.0.0）
3. 记录需求列表和功能说明
4. 记录项目结构和文件说明

### 场景2：代码修改后
当用户修改代码后：
1. 自动递增版本号（根据修改类型）
2. 更新README中的版本号
3. 记录本次修改的内容
4. 更新变更日志（CHANGELOG）

### 场景3：新功能添加
当添加新功能时：
1. 递增次版本号（如v1.0.0 → v1.1.0）
2. 在README中添加新功能说明
3. 更新使用示例
4. 更新变更日志

## 版本号显示位置

### 推荐位置
1. **工具名称旁边**：在标题或工具名称后显示版本号
   - 示例：`工具名称 v1.0.0`
2. **右下角**：在页面或窗口的右下角显示版本号
   - 样式：小字号、浅灰色、不显眼

### 显示样式
- 字体大小：10-12px
- 颜色：#999999（浅灰色）
- 位置：右下角或名称旁
- 格式：`v{主版本}.{次版本}.{修订号}`

## 工作流程

### 步骤1：需求确认
1. 与用户确认所有需求
2. 创建或更新README.md
3. 设置初始版本号为v1.0.0
4. 记录需求列表

### 步骤2：开发过程中
1. 每次修改代码后：
   - 判断修改类型（新功能/bug修复/优化）
   - 递增相应的版本号
   - 更新README中的版本号
   - 在变更日志中记录本次修改

### 步骤3：功能完成
1. 确认所有功能已实现
2. 更新README中的使用说明
3. 添加示例和截图（如适用）
4. 完善文档

### 步骤4：发布准备
1. 检查README完整性
2. 验证版本号正确性
3. 确认变更日志完整
4. 在工具界面显示版本号

## 版本号递增规则

### 主版本递增（X.0.0 → Y.0.0）
- 重大功能变更
- 不兼容的API修改
- 架构性重构

### 次版本递增（1.X.0 → 1.Y.0）
- 新功能添加
- 重大改进
- 向后兼容的API扩展

### 修订号递增（1.0.X → 1.0.Y）
- bug修复
- 小的改进
- 文档更新
- 性能优化

## 注意事项

1. **始终更新README**：每次修改代码后都要更新
2. **版本号可见但不显眼**：放在工具名称旁或右下角
3. **变更日志要详细**：记录每次修改的具体内容
4. **保持一致性**：使用统一的版本号格式和文档结构
5. **及时更新**：不要等到最后才更新，每次修改后立即更新
