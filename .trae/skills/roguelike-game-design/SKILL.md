# 肉鸽游戏策划师

## 技能概述

本技能专注于为开发肉鸽(Roguelike/Roguelite)类型游戏的开发者提供策划和设计专业支持。肉鸽游戏以随机生成、永久死亡、挑战性等元素为特点，本技能将帮助开发者设计出有趣、平衡且引人入胜的肉鸽游戏体验。

## 适用场景

- 设计通用肉鸽游戏的核心玩法和机制
- 规划游戏关卡和地图设计
- 设计道具和装备系统
- 策划敌人和Boss设计
- 平衡游戏难度和 progression 系统
- 设计游戏叙事和主题
- 解决肉鸽游戏设计中的常见问题

## 示例用法

### 示例1：核心玩法设计文档

```markdown
# 肉鸽游戏核心玩法设计文档

## 游戏概念
- **游戏名称**：《深渊迷途》
- **游戏类型**：Roguelite动作角色扮演
- **核心机制**：
  - 程序化地牢生成
  - 永久死亡
  - 角色升级和技能树
  - 随机道具和装备
  - 动态难度调整

## 游戏循环
1. **开始**：玩家选择角色和初始装备
2. **探索**：在随机生成的地牢中探索
3. **战斗**：与敌人和Boss战斗
4. **成长**：收集经验和道具，升级角色
5. **死亡**：失去所有当前进度，返回基地
6. ** progression**：使用解锁的资源和能力开始新的冒险

## 核心系统

### 1. 角色系统
- **基础属性**：生命值、法力值、攻击力、防御力、速度
- **技能树**：三大分支（战士、法师、游侠）
- **天赋系统**：可解锁的被动能力

### 2. 道具系统
- **品质等级**：普通、优秀、稀有、史诗、传奇
- **道具效果**：主动技能、被动加成、状态效果
- **道具组合**：特定组合触发额外效果

### 3. 地牢系统
- **主题区域**：森林、洞穴、城堡、地狱
- **房间类型**：普通、精英、宝藏、商店、祭坛
- **环境危险**：陷阱、障碍物、环境效果

### 4. 敌人系统
- **敌人类型**：普通、精英、Boss
- **行为模式**：巡逻、埋伏、追击、协作
- **特殊能力**：远程攻击、近战攻击、 buff、 debuff

### 5. 成长系统
- **经验和等级**：击败敌人获得经验
- **技能点数**：升级获得，用于解锁技能
- **解锁系统**：永久解锁新角色、道具、能力
```

### 示例2：关卡生成设计

```markdown
# 程序化关卡生成设计

## 生成算法

### 1. 房间生成
- **房间大小**：随机 3x3 到 8x8 格子
- **房间数量**：基于当前深度（1-5层 8-12个房间，6-10层 12-16个房间）
- **房间类型**：
  - 普通房间 (60%)
  - 精英房间 (15%)
  - 宝藏房间 (10%)
  - 商店房间 (10%)
  - 特殊房间 (5%)

### 2. 走廊连接
- **连接方式**：使用 BSP (Binary Space Partitioning) 算法
- **走廊宽度**：1-2格子
- **走廊形状**：直线或L形

### 3. 主题应用
- **主题选择**：基于当前区域（森林、洞穴、城堡等）
- **视觉元素**：墙壁、地板、装饰的主题化
- **环境效果**：主题相关的环境危险和益处

### 4. 内容填充
- **敌人放置**：基于房间类型和当前深度
- **道具放置**：基于房间类型和随机因素
- **地形特征**：随机添加障碍物、陷阱、互动元素

### 5. 连通性保证
- **路径验证**：确保所有房间都可到达
- **入口/出口**：每个关卡有明确的起点和终点
- **捷径**：后期关卡可能包含回到起点的捷径
```

### 示例3：道具系统设计

```markdown
# 道具系统设计

## 道具分类

### 1. 武器
- **近战武器**：剑、斧、锤、匕首等
- **远程武器**：弓、弩、火枪、魔法杖等
- **特殊武器**：双手武器、双持武器

### 2. 防具
- **头盔**：提供头部保护和特殊效果
- **胸甲**：提供身体保护和特殊效果
- **手套**：提供手部保护和特殊效果
- **靴子**：提供脚部保护和移动加成

### 3. 饰品
- **戒指**：提供各种被动效果
- **项链**：提供各种被动效果
- **徽章**：提供职业相关的特殊效果

### 4. 消耗品
- **药水**：恢复生命值、法力值、提供临时 buff
- **卷轴**：提供一次性魔法效果
- **炸弹**：造成范围伤害和破坏

### 5. 特殊物品
- **钥匙**：解锁特定区域
- **符文**：解锁技能或提供永久加成
- **圣物**：提供强大的被动效果

## 稀有度系统

| 稀有度 | 颜色 | 出现概率 | 效果强度 |
|--------|------|----------|----------|
| 普通 | 白色 | 60% | 基础效果 |
| 优秀 | 绿色 | 25% | 增强效果 |
| 稀有 | 蓝色 | 10% | 强大效果 |
| 史诗 | 紫色 | 4% | 独特效果 |
| 传奇 | 橙色 | 1% | 传说效果 |

## 道具生成规则

1. **基础池**：每个稀有度有对应的道具池
2. **权重系统**：基于玩家等级和当前深度调整道具权重
3. **职业倾向**：根据玩家角色职业调整道具生成倾向
4. **协同效果**：生成道具时考虑与玩家现有装备的协同效果
5. **唯一性**：传奇道具保证唯一性，不会重复生成
```

## 最佳实践

1. **核心循环设计**：确保游戏有清晰、有趣的核心循环，让玩家有重复游玩的动力
2. **随机与控制平衡**：在随机性和玩家可控性之间找到平衡，避免过度随机导致的挫败感
3. **渐进式难度**：设计合理的难度曲线，让玩家随着经验增长感受到挑战
4. **有意义的选择**：为玩家提供有意义的游戏选择，增强游戏的策略性
5. **持续的奖励**：设计短期和长期的奖励机制，保持玩家的动力
6. **测试和迭代**：通过大量测试和玩家反馈不断调整游戏设计
7. **主题一致性**：确保游戏的各个元素都符合整体主题和风格
8. ** accessibility**：考虑不同技能水平的玩家，提供适当的难度选项

## 常见问题

### Q: 如何平衡随机性和可控性？

A: 可以使用以下方法：
- 实现种子系统，让玩家可以重现特定的游戏体验
- 设计"安全网"机制，确保玩家不会连续遇到过于困难的情况
- 提供玩家可以控制的资源和选择，如商店、升级系统
- 使用加权随机系统，控制稀有和强力内容的出现频率
- 实现基于玩家表现的动态难度调整

### Q: 如何设计有意义的永久死亡机制？

A: 设计建议：
- 实现 meta progression 系统，即使死亡也能获得一些长期 progression
- 设计基地或 hub 区域，让玩家可以在冒险之间进行准备和规划
- 提供不同难度选项，包括可以关闭永久死亡的模式
- 确保死亡有明确的原因和教训，让玩家能够从失败中学习
- 设计死亡动画和反馈，让死亡成为游戏体验的一部分

### Q: 如何保持游戏的长期 replayability？

A: 解决方案：
- 实现多样化的角色和职业选择
- 设计丰富的道具和装备系统，提供多种 build 可能性
- 创建程序化生成的关卡和内容，确保每次游戏体验不同
- 设计挑战和成就系统，鼓励玩家尝试不同的游戏风格
- 定期更新和添加新内容，保持游戏的新鲜感

### Q: 如何设计平衡的 Boss 战？

A: 设计建议：
- 为每个 Boss 设计独特的机制和阶段
- 确保 Boss 有明确的攻击模式和弱点
- 设计适当的难度曲线，让 Boss 战既有挑战性又可战胜
- 提供 Boss 战的视觉和听觉反馈，增强紧张感
- 确保 Boss 战有适当的奖励，与挑战难度匹配

### Q: 如何设计有效的 progression 系统？

A: 建议：
- 区分短期 progression（单局游戏内）和长期 progression（跨游戏）
- 设计清晰的解锁路径和目标
- 确保 progression 提供有意义的力量增长和选择
- 平衡 progression 速度，避免过快或过慢
- 为不同类型的玩家提供不同的 progression 路径
